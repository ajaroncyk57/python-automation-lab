# Restaurant POS Sales Reporting

A small Python point-of-sale (POS) and Sales Reporting project inspired by an early programming exercise.

## Project Status

This project is currently a rebuild candidate. The original version demonstrates a menu-based customer order flow and basic end-of-day reporting. The rebuild will focus on cleaner code structure, transaction tracking, and sales analytics.

## Business Problem

Small restaurants need a simple way to capture orders, calculate totals, and summarize sales activity. This project explores how a lightweight Python application can support basic transaction and reporting workflows.

## Current Features

- Captures customer burger orders.
- Calculates item-level order totals.
- Tracks selected menu items.
- Produces a basic end-of-day sales report.

## Planned Improvements

- Refactor menu items into structured data.
- Track each order as a transaction.
- Export orders to CSV.
- Add item/category-level sales reporting.
- Add receipt formatting.
- Add reusable functions and input validation.
- Add unit tests.
